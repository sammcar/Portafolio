# Security Policy

## Reporting a vulnerability

If you find a security issue in this site (dependency vulnerability, XSS via
MDX content, leaked secret in history, etc.), please **do not open a public
issue**. Use GitHub's private vulnerability reporting
("Security" tab → "Report a vulnerability") or contact me through the site.

You can expect an acknowledgment within a few days.

## Scope

The deployed site and this repository. Dependency alerts are handled
automatically via Dependabot + `npm audit` in CI.
